# Refactor SwiftUI View

## Scenario

An iOS developer asks a model to refactor a large SwiftUI view.

## Prompt

Refactor a large SwiftUI view into smaller components without changing behavior.

## Expected Behavior

- Preserves behavior.
- Improves readability.
- Uses appropriate private helper views or subviews.
- Avoids unnecessary abstractions.

## Scoring Rubric

Evaluate behavioral preservation, SwiftUI idiom, readability, and integration cost.

## Notes for Reviewers

Review generated diffs rather than isolated snippets when possible.
