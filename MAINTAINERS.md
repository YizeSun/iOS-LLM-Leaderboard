# Maintainers

## Release maintainers

- [`@YizeSun`](https://github.com/YizeSun) — project owner and current release
  maintainer.

## Authority

Release maintainers may merge repository changes, accept evidence transitions,
record corrections or withdrawals, deprecate evidence, and approve releases.
Automation and AI-assisted review may provide findings, but they do not replace
a maintainer's recorded decision.

An explicit maintainer instruction is required before publishing a GitHub
Release, creating a benchmark version tag, enabling official-result status, or
authorizing default leaderboard ranking.

## Conflicts

Maintainers disclose material relationships to a submitted model, runtime,
hardware vendor, or result. When another maintainer is available, a conflicted
maintainer should not be the only reviewer. Maintainer-run evidence remains
permitted but must use the `maintainer-reference` label and the same integrity,
privacy, and metric-eligibility rules as community evidence.
