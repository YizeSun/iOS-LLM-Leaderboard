# Privacy and Compliance Methodology

Privacy and compliance reviews evaluate whether a model or provider is suitable for App Store-facing iOS applications.

This project does not provide legal advice. Reviews should cite public provider documentation and clearly separate facts from interpretation.

## Evaluation Items

- provider privacy policy
- whether user data is used for training
- data retention controls
- logging controls
- safety guardrails
- suitability for App Store-facing applications
- local or offline inference support
- enterprise or zero-retention options where relevant

## Review Format

Each review should include:

- provider and model name
- documentation sources reviewed
- date reviewed
- known limitations
- open questions

Do not infer undisclosed provider behavior.
